# 项目启动完整指南

## 🚀 推荐方式：使用 Docker 一键启动

由于手动启动前端服务遇到问题，强烈建议使用 Docker 方式启动整个项目。

### 步骤 1: 配置 GLM API Key

编辑 `backend/.env` 文件：
```bash
nano backend/.env
```

找到这一行并填入你的 API Key：
```env
OPENAI_API_KEY="your-glm-api-key-here"
```

### 步骤 2: 启动所有服务

```bash
cd /Users/robinxie/01-开发项目/AI\ 智能简历
docker-compose up -d
```

这会自动启动：
- ✅ PostgreSQL 数据库
- ✅ Redis 缓存
- ✅ 后端 API 服务
- ✅ 前端开发服务器

### 步骤 3: 等待服务启动

首次启动需要构建镜像，大约 5-10 分钟。查看进度：
```bash
docker-compose logs -f
```

### 步骤 4: 访问应用

服务启动完成后：
- 前端：http://localhost:5173
- 后端 API：http://localhost:8000
- API 文档：http://localhost:8000/docs

---

## 🔧 如果 Docker 方式也不行

### 方案 A: 检查 Docker 状态
```bash
# 检查 Docker 是否运行
docker ps

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs frontend
```

### 方案 B: 重新构建
```bash
# 停止所有服务
docker-compose down

# 重新构建并启动
docker-compose up -d --build
```

### 方案 C: 单独启动前端容器
```bash
docker-compose up frontend
```

## 📝 当前问题分析

根据错误 `ERR_CONNECTION_REFUSED`，问题是：
1. **前端服务器没有启动** ← 最可能
2. 端口被占用
3. 防火墙阻止

**解决方案**：使用 Docker 可以避免所有这些问题，因为容器内部环境是隔离和预配置好的。
