# AI 智能简历优化应用 - 实施计划

## 一、项目概览

### 1.1 项目信息
- **项目名称**：AI 智能简历优化应用
- **开发周期**：8-10 周
- **技术栈**：React + TypeScript + FastAPI + PostgreSQL
- **部署方式**：Docker + Nginx

### 1.2 核心目标
1. 实现简历智能解析和结构化存储
2. 实现职位需求智能分析
3. 实现简历与职位的精准匹配
4. 实现 AI 驱动的简历优化
5. 提供专业的简历导出功能

### 3.3 数据库实施

#### 3.3.1 核心表设计（SQL）

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 简历表
CREATE TABLE resumes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    title VARCHAR(100) NOT NULL,
    file_path VARCHAR(255),
    file_type VARCHAR(20),
    parsed_data JSONB,
    is_default BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 职位表
CREATE TABLE job_positions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    title VARCHAR(200) NOT NULL,
    company VARCHAR(100),
    description TEXT,
    requirements JSONB,
    source_url VARCHAR(500),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 匹配记录表
CREATE TABLE match_records (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    resume_id UUID REFERENCES resumes(id) ON DELETE CASCADE,
    job_id UUID REFERENCES job_positions(id) ON DELETE CASCADE,
    match_score DECIMAL(5,2),
    match_details JSONB,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 优化简历表
CREATE TABLE optimized_resumes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    original_resume_id UUID REFERENCES resumes(id) ON DELETE CASCADE,
    job_id UUID REFERENCES job_positions(id) ON DELETE CASCADE,
    optimized_content JSONB,
    optimization_suggestions JSONB,
    file_path VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_resumes_user_id ON resumes(user_id);
CREATE INDEX idx_jobs_user_id ON job_positions(user_id);
CREATE INDEX idx_match_resume_id ON match_records(resume_id);
CREATE INDEX idx_match_job_id ON match_records(job_id);
CREATE INDEX idx_optimized_original_id ON optimized_resumes(original_resume_id);
```
