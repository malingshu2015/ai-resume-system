# 职位文档上传功能 - 快速开始

## 快速测试（5 分钟）

### 1. 准备测试文档

创建一个简单的测试文档 `test-job.txt`，然后转换为 PDF 或 Word：

```
高级 Python 后端工程师

公司：某科技有限公司
地点：北京 · 朝阳区
薪资：25K-40K

职位描述：
我们正在寻找一位经验丰富的 Python 后端工程师加入我们的团队。

岗位职责：
1. 负责后端服务的设计、开发和维护
2. 参与系统架构设计和技术选型
3. 优化系统性能，提升用户体验

任职要求：
1. 本科及以上学历，计算机相关专业
2. 3年以上 Python 开发经验
3. 熟悉 FastAPI、Django 等 Web 框架
4. 熟悉 PostgreSQL、Redis 等数据库
5. 具有良好的代码规范和文档习惯
```

### 2. 启动应用

```bash
# 终端 1：启动后端
cd backend
uvicorn app.main:app --reload

# 终端 2：启动前端
cd frontend
npm run dev
```

### 3. 测试功能

1. **打开浏览器**
   - 访问 http://localhost:5173

2. **导航到录入页面**
   - 点击"录入新职位"或直接访问 `/job-input`

3. **选择文档上传标签页**
   - 点击"文档上传解析"标签

4. **上传测试文档**
   - 点击上传区域，选择刚才创建的 PDF 或 Word 文件
   - 或者直接拖拽文件到上传区域

5. **查看结果**
   - 等待 3-10 秒
   - 系统自动跳转到"手动输入/确认"标签页
   - 检查提取的信息是否正确

6. **提交录入**
   - 点击"提交录入进入解析"按钮
   - 查看成功页面

## 常见问题快速解决

### Q1: 上传后显示"文件类型不支持"
**解决**: 确保上传的是 .pdf、.doc 或 .docx 格式的文件

### Q2: 显示"文档解析失败"
**解决**: 
- 检查文档是否为扫描版（图片格式）
- 尝试重新导出文档为标准格式
- 或使用"截图识图分析"功能

### Q3: 提取的信息不准确
**解决**: 在"手动输入/确认"页面手动修正信息

### Q4: 上传后一直加载
**解决**:
- 检查后端服务是否正常运行
- 查看浏览器控制台是否有错误
- 查看后端日志
